# 01 — Summary

## Recommendation

Present the experience as **one shared data platform with three modes**, rather than three separate destinations.

Recommended primary choices:

1. **Contribute & Build**
2. **Explore & Analyze**
3. **Published Results**

These labels are preferable to “Dashboard / Portal / Presentation” because they describe **user intent** rather than internal system structure.

## Why this framing works

The platform supports three distinct intents:

- **Operational intent** — contributors and developers ingest, maintain, and monitor data systems
- **Investigative intent** — analysts and readers query, filter, and explore shared datasets
- **Communication intent** — stakeholders consume curated, presentation-ready outputs

If presented as three disconnected pages, users may ask:
- Which one contains the real data?
- Why are there separate places if the datasets are the same?
- Which one should I start in?

A unified homepage reduces this ambiguity.

## UX direction

Create a **GitHub-style logged-in home/dashboard** with:
- a shared top navigation
- a central task-based entry area
- utility-oriented card modules
- contextual sidebars
- recent activity and featured content

This should feel like a working product interface, not a marketing homepage.

## Preferred navigation labels

Recommended:
- **Home**
- **Workspace**
- **Explore**
- **Published Results**
- **Datasets**
- **Docs**

Where:
- **Workspace** serves contributors/developers
- **Explore** serves readers/analysts
- **Published Results** serves presentation audiences

## Core message to reinforce

Because both the dashboard and portal are backed by the same datasets, the homepage should make this explicit:

> The same trusted datasets power data contribution, analysis, and published results.

## Primary design goal

Help users answer:

**What do I want to do today?**

not:

**Which system am I supposed to use?**

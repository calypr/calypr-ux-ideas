# 04 — Content Copy

## Homepage title and intro

### Option A
**One platform for contributing, exploring, and publishing data**  
Use the same trusted datasets to manage submissions, perform analysis, and share published findings.

### Option B
**Work with shared data from submission to publication**  
Contribute data, explore it in depth, and present approved results from one connected platform.

### Option C
**Choose your workspace**  
Access the same trusted datasets through tools for data contribution, analysis, and published results.

## Primary cards

### Contribute & Build
Label:
_For contributors and developers_

Description options:
- Submit data, manage workflows, and monitor pipeline health.
- Manage ingestion, maintain quality, and track system activity.
- Contribute data and keep operational workflows running smoothly.

CTA options:
- **Open workspace**
- **Go to workspace**
- **Manage data operations**

### Explore & Analyze
Label:
_For readers and analysts_

Description options:
- Search, query, filter, compare, and export shared datasets.
- Explore the same datasets through interactive analysis tools.
- Investigate data, save queries, and generate analytical outputs.

CTA options:
- **Start exploring**
- **Open Explore**
- **Query data**

### Published Results
Label:
_For stakeholders and collaborators_

Description options:
- Browse curated findings and presentation-ready outputs.
- View approved analyses, released summaries, and published results.
- Access curated, stable views of findings for sharing and review.

CTA options:
- **View published results**
- **Open results**
- **Browse findings**

## Shared data message

- **The same datasets power Workspace, Explore, and Published Results.**
- **Contribute, analyze, and publish from one shared data foundation.**
- **Move from data operations to analysis to publication without leaving the platform.**

## Navigation labels

Recommended:
- Home
- Workspace
- Explore
- Published Results
- Datasets
- Docs

Avoid:
- Dashboard
- Portal
- Presentation
- Viewer

Reason:
These labels describe internal constructs more than user value.

## Tone guidelines

Use language that is:
- clear
- concrete
- task-focused
- calm
- non-promotional

Prefer:
- “Query datasets”
- “View published results”
- “Manage workflows”

Avoid:
- “Unlock insights”
- “Leverage data ecosystems”
- “Empower stakeholders”

## Microcopy guidance

### Good examples
- Open workspace
- Start exploring
- View results
- Recent datasets
- Saved queries
- Featured publication

### Avoid
- Enter portal
- Launch presentation mode
- Open dashboard environment

## Overall copy principle

Help users understand:
- what each area is for
- who it is for
- how it connects to the same data

Do not rely on internal product vocabulary to do that work.
